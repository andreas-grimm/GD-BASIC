3   /**
4    * This defines the different states the tokenizer can be in while it's
5    * scanning through the source code. Tokenizers are state machines, which
6    * means the only data they need to store is where they are in the source
7    * code and this one "state" or mode value.
8    *
9    * One of the main differences between tokenizing and parsing is this
10   * regularity. Because the tokenizer stores only this one state value, it
11   * can't handle nesting (which would require also storing a number to
12   * identify how deeply nested you are). The parser is able to handle that.
13   */
14  public enum TokenizeState {
15      DEFAULT, WORD, NUMBER, STRING, COMMENT
16  }
